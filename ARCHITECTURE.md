## Видение проекта

Браузерная тактическая RPG, вдохновлённая **Final Fantasy Tactics**, построенная на **Phaser 4 + TypeScript + Vite**.  
Внешних арт-ассетов нет — вся графика рисуется процедурно через Phaser Graphics API.

### ✅ Реализовано
- Scaffold: Phaser 4 + Vite + TypeScript; репозиторий `kuirsan26-lab/fft-phaser`
- Изометрический рендерер сетки (10×8, 4 типа тайлов, высота 0–3, алгоритм художника)
- CT-система ходов, BFS-поиск пути с учётом высоты и прыжка
- Боевая система: физ/маг урон, лечение, AoE, стан, провок; бонус высоты ×1.15
- Способности: Attack, Shield Bash, Provoke, Fire, Blizzard, Thunder, Heal, Shoot, Rain of Arrows
- 4 класса: Воин, Рыцарь, Маг, Лучник (лучник — только дальний бой)
- MP-реген каждый ход (по классу: маг 8, остальные 4–5)
- ИИ: приоритет способностей → ближний бой по слабейшему → движение; провокованный атакует только провокатора
- UI: меню действий, панель ходов, тултип, лог, HP-бары (цвет по уровню HP), числа урона
- Автобой (клавиша A), зона опасности врага при наведении
- **Режим кампании:** создание отряда → 10 битв → лагерь → победа
  - Перманентная смерть, масштабирование врагов (×0.7 → ×1.7), HP/MP переносятся
- Фиксы рендеринга: корректные центры тайлов, depth-порядок оверлеев
- **Система уровней:** XP за каждую битву (50+10×N), прокачка главного стата по классу (+2 ATK/DEF/MAG или +1 ATK+SPD), уровень и XP-бар в лагере

## 🔥 Приоритет 2 — Roguelite-петля (следующий)

**Проблема:** игрок выигрывает 10 одинаковых битв на одной карте, юниты между ними не растут.  
Без нарратива роста перманентная смерть — просто неудобство, а не трагедия (Fire Emblem, FFT).

- [x] **Система уровней** — после битвы выжившие получают XP; при накоплении — уровень (+2 к главному стату класса: воин/рыцарь → ATK/DEF, маг → MAG, лучник → ATK+SPD). Отображать в `CampScene`.
- [ ] **AI по ролям** — маг держит дистанцию ≥3, лучник не входит в ближний бой, раненый (<30% HP) отступает, рыцарь идёт к раненому союзнику-магу. Делает каждый бой тактически другим.
- [ ] **Эффекты местности** — лес: 25% шанс промаха входящей атаки. Одна строка в формуле урона, но игрок начинает думать о позиции.

## Архитектура

```
src/
├── main.ts               — конфиг Phaser, регистрация сцен
├── scenes/
│   ├── BootScene.ts           — заставка → SquadCreationScene
│   ├── SquadCreationScene.ts  — имя отряда + выбор 4 классов, запуск кампании
│   ├── BattleScene.ts         — основной игровой цикл; поддерживает кампанию
│   ├── CampScene.ts           — лагерь: ростер, лечение, прогресс, (P2: уровни)
│   └── UIScene.ts             — параллельная сцена: заголовок, ход, оверлей конца боя
├── systems/
│   ├── GridSystem.ts     — изометрический рендеринг, подсветка, координаты
│   ├── TurnManager.ts    — CT-очередь, MP-реген, tickStatuses
│   ├── Pathfinder.ts     — BFS-движение, расчёт дальности, восстановление пути
│   └── CombatSystem.ts   — урон, способности, бонус высоты, ИИ, провок
├── entities/
│   └── Unit.ts           — статы, HP/MP/mpRegen, статусы (stun, provoked), спрайт
├── data/
│   ├── UnitData.ts       — шаблоны классов, все 9 способностей
│   ├── MapData.ts        — карта высот, позиции спавна
│   └── Campaign.ts       — CampaignState, состав врагов по битве, скейлинг
└── ui/
    ├── ActionMenu.ts     — меню (Движение / Способности / Ожидание)
    ├── TurnOrderPanel.ts — полоса предстоящих ходов с CT-барами
    └── BattleLog.ts      — лог боя + всплывающие числа урона
```

## Ключевые архитектурные решения

### CT-система (Charge Time)
`ct` растёт на `spd` за тик. При `ct >= 100` юнит действует (`ct -= 100`). В начале хода: `tickStatuses()` → `regenMp()`.

### Изометрическая проекция
```
screen_x = (col - row) * TILE_W/2 + offsetX
screen_y = (col + row) * TILE_H/2 + offsetY - height * TILE_DEPTH
```
Тайлы и оверлеи отсортированы по `col + row`. Оверлеи вставляются через `container.addAt` сразу после базового тайла — рельеф корректно их перекрывает.

### Коммуникация между сценами
`BattleScene` → события `turnStart`, `battleOver`, `campaignInfo`. `UIScene` слушает через `scene.get('BattleScene').events.on(...)`. Данные кампании — через `scene.start(name, data)` (технический долг: перевести на `game.registry`).

### Архитектура ИИ (`CombatSystem.aiChooseAction`)
1. Если провокован → атаковать только провокатора
2. Использовать лучшую доступную способность в радиусе
3. Ближний бой по слабейшему врагу
4. Двигаться к ближайшему врагу

**P2:** добавить роль-ориентированное поведение и отступление по HP.

### Формулы боя
- Физ. урон: `(atk × 1.8 × dmgMult − def × 0.5 ± rand(6)) × heightBonus`
- Маг. урон: `(mag × 2.5 × dmgMult − targetMag × 0.3 ± rand(6)) × heightBonus`
- Лечение: `mag × 3 × healMult`
- `heightBonus = 1.15` если атакующий выше цели, иначе `1.0`
- **P2:** лес → 25% шанс промаха

## Запуск локально

```bash
npm install
npm run dev    # http://localhost:5173
npm run build  # dist/
```

## Управление

| Ввод | Действие |
|------|----------|
| Клик по юниту | Выбор / информация о юните |
| Клик по синему тайлу | Движение |
| Клик по красному тайлу | Выбор цели способности |
| Наведение на врага | Показать оранжевую зону его атаки |
| ESC | Отмена / назад в меню |
| R | Перезапуск боя (только в свободном режиме) |
| A | Переключить автобой |
